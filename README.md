# 医学教材文本切分方法对比实验  
## 面向医学 RAG 系统的文本切分策略研究

## 项目简介

在 **检索增强生成（RAG, Retrieval-Augmented Generation）** 系统中，文本切分（Chunking）直接影响检索召回率与问答质量。然而，不同文本结构适用的切分策略并不相同。

医学教材具有以下特点：

- 知识点 **篇幅长**
- 专业术语 **密集**
- 知识结构 **层级化明显**
- 跨句/跨段落 **逻辑依赖强**

这些特点使得通用文本切分策略在医学领域往往表现不稳定。

本项目基于 **医学教材文本**，对比三种主流文本切分方法，并提出一种 **RecursiveMaxMin 混合切分策略**，通过统一评估框架分析不同策略在医学 RAG 检索任务中的表现。

本实验重点研究：

**不同文本切分策略对 RAG 检索性能的影响。**

---

# 研究问题

本项目主要关注以下问题：

1. 不同文本切分方法在医学教材上的效果差异  
2. 语义切分是否在医学长文本中仍然具有优势  
3. 是否存在更适合医学教材结构的混合切分策略  

---

# 切分方法对比

本实验共实现并对比四种文本切分方法：

## 1 固定长度切分（Fixed Chunking）

按照固定字符长度进行切分。

特点：

- 实现简单  
- 长度分布稳定  
- 可能截断句子  

---

## 2 递归结构切分（Recursive Chunking）

按照文本结构逐级切分：

```
段落 → 句子 → 标点
```

优点：

- 尽量保持语义边界  
- 不破坏句子结构  

---

## 3 传统语义切分（Semantic Chunking）

基于句向量相似度进行动态合并：

```
句子 embedding
↓
相邻句子相似度
↓
低于阈值时切分
```

理论优势：

- 语义一致性强  

缺点：

- 容易产生大量小碎片  

---

## 4 RecursiveMaxMin 混合切分（本项目方法）

本项目提出一种 **结构 + 语义结合的混合切分方法**。

整体流程：

```
递归结构粗切分
        ↓
语义相似度合并
        ↓
长度归一化
```

核心目标：

- 保留文本结构信息  
- 避免语义碎片化  
- 保证 chunk 长度适合 embedding 模型  

适用于：

**医学教材 / 技术文档 / 长知识点文本**

# 评估指标

## 检索指标

### HitRate@3

Top3 检索结果中包含正确答案的比例。

```
HitRate@3 = 命中问题数 / 总问题数
```

---

### MRR（Mean Reciprocal Rank）

正确答案排名倒数的平均值。

```
MRR = mean(1 / rank)
```

该指标反映：

**正确文档在检索结果中的平均排序质量。**

---

## 文本结构指标

为分析不同切分方法的结构质量，本项目额外统计：

| 指标 | 含义 |
|----|----|
| 块数量 | 总 chunk 数量 |
| 平均长度 | 每个 chunk 平均字符数 |
| 标准差 | chunk 长度离散程度 |
| 碎片数量 | 小于100字符的块 |
| 超长块 | 大于800字符的块 |
| 完整句比例 | 句子未被截断比例 |
| 语义连贯性 | 相邻句语义相似度 |

这些指标用于分析：

**结构质量对检索性能的影响。**

# 结果分析

### 1 语义切分不适合医学教材

语义切分产生大量短文本：

```
平均长度 ≈ 100
碎片数极高
```

导致：

- 知识点被过度拆分  
- 检索召回率下降  

---

### 2 递归结构切分表现稳定

递归切分利用教材的结构特征：

```
章节
段落
句子
```

能够保持较完整知识块。

---

### 3 RecursiveMaxMin 在部分任务中取得最佳结果

在 **内科学问答任务** 中：

- HitRate@3  
- MRR  

均达到最高。

原因：

- 结合结构信息与语义信息  
- 避免过度碎片化  
- chunk 长度更适合 embedding 模型  

---

### 4 外科学文本更适合长 chunk

外科学教材中：

- 单知识点更长  
- 段落信息密度更高  

在该场景下：

```
较长 chunk
```

反而更有利于检索。

---
